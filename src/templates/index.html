NophiGene Gene Workbench

This workspace is built for red-biotech style gene analysis: interval discovery, methylation preprocessing, variant review, and concise artifact generation around the selected locus. The overview stays separate so the active work surfaces can remain dense, wide, and practical once you start working with real data.

What this workbench expects

The app reads local project inputs from {{ data_dir }} and writes reports to {{ results_dir }}. It scans for VCFs, paired IDAT prefixes, and optional population statistics files so the browser stays close to the real project layout.

Preprocessing layer: resolve a gene interval and create a gene-specific EPIC manifest subset.
Variant layer: sequence calls from a regional VCF focused on the selected gene interval.
Epigenetic layer: methylation beta values derived from paired Illumina IDAT files and the filtered manifest subset.

Why the workflow is split into tabs

The overview explains the biological workflow once. Preprocessing resolves the region and builds the methylation subset first. Only then does the analysis tab open, so the main analysis surface is always tied to a prepared gene interval and ready-to-use manifest subset.

Designed to keep the active work surface compact, high-density, and stepwise.

Runtime map

How NophiGene Is Wired

Both launch modes use the same gene-analysis core. The local path runs directly from the project virtualenv, while the Docker path wraps the same Flask UI and Python pipeline inside a container with mounted input and output folders.

Local runtime flow

Direct Windows launch

Use this path for the fastest startup and the most convenient day-to-day work on the same machine.

User browser Open http://127.0.0.1:8000 after the local starter script launches the UI.
↓
Starter icon and local Python Start NophiGene UI.cmd calls the PowerShell launcher, which starts .venv\Scripts\python.exe.
↓
Web layer src/app.py boots the Flask app in src/webapp.py and renders the red-biotech workspace.
↓
Analysis core src/analysis.py loads PASS variants, processes IDAT methylation data, applies gene-specific manifest subsets, and builds the previews.
↓
Inputs data/ stores the VCF, IDATs, sample sheet, and related run artifacts.
Bundled knowledge src/gene_data/ provides the bundled knowledge bases and prepared helper data that ship with the app.
Outputs results/ receives the report HTML and methylation CSV exported by the pipeline.
Best when: You want the shortest startup time, easy access to local files, and no container rebuild while iterating on the app.
Docker runtime flow

Containerized launch

Use this path when you want a more reproducible environment or need to share a controlled runtime setup.

User browser Connect to the same UI endpoint on http://127.0.0.1:8000, but served from the container.
↓
Docker starter path Start NophiGene UI (Docker).cmd starts Docker Desktop if needed, builds the image, and runs the NophiGene container.
↓
Container runtime The image starts the same Flask layer and Python pipeline, with port 8000 published back to the host machine.
↓
Mounted inputs The host data/ folder is mounted into the container so local VCF and IDAT files stay outside the image.
Bundled app code src/ and src/gene_data/ are copied into the image so the app, bundled knowledge bases, and preprocessing helpers travel together.
Mounted outputs The host results/ folder is mounted back out so generated reports persist after the container stops.
Best when: You want a cleaner, reproducible runtime boundary and are comfortable with the extra startup cost of Docker builds.
